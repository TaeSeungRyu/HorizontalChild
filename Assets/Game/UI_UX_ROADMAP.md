# UI / UX 폴리시 로드맵

게임 시스템 작업과 시각 폴리시를 병행하기 위한 가이드.

---

## 원칙

### 🟢 지금 해도 좋은 것 (변경 비용 적음 + 즉각 효과)
- 바다 시각화 — 셰이더/텍스처 교체
- 육지 색상·디테일 — 단색 → 지역별 톤
- 패널 아이콘·구분선 — 텍스트 옆 이모지·SVG, 카드형 배경

### 🟡 시스템 안정 후가 좋은 것
- 패널 레이아웃 전면 재설계 — 구조 자주 바뀌면 폴리시 낭비
- 튜토리얼 일러스트 — 게임플레이 흐름 확정 후

### 🔴 마지막에 하는 것
- 애니메이션 트랜지션 (페이드, 슬라이드)
- 사운드 · BGM
- 파티클 · 이펙트 풀세트

---

## 어린이 게임 특수성

일반 게임보다 시각 폴리시를 **일찍** 해야 하는 이유:

- placeholder 큐브 / 단색으로 어린이 테스트하면 흥미 잃음 → 피드백 신뢰도 ↓
- 시각 매력 자체가 게임의 가치
- "어린이 대항해시대" 컨셉이라면 대항해시대2 시대의 색감 · 낭만이 핵심 자산

→ 바다 · 육지는 **지금 손대도 OK**. 패널은 구조 fix 된 것부터.

## 추천 워크플로우 (5주 사이클 예시)

| 주차 | 시스템 작업 | 시각 폴리시 |
|---|---|---|
| 1 | 큰 지도 / 카메라 줌 | — |
| 2 | 게임일 시스템 | **바다 셰이더** |
| 3 | Phase 4 전투 폴리시 | **육지 컬러 팔레트** |
| 4 | 명성 게이트 (해상 NPC) | **패널 폴리시 (안정 패널부터)** |
| 5 | 튜토리얼 보강 | 튜토리얼 일러스트 |

**핵심**: 시스템 + 시각을 **번갈아**. 한쪽만 계속 하면 지침. 시각이 점진적으로 좋아져야 테스트 세션마다 피드백 품질 ↑.

## 즉시 작업 후보

### A. 바다 셰이더 (1~2시간) ⭐ 추천
**임팩트**: 게임 켜자마자 보이는 화면 절반. 가장 큰 시각 효과.

**옵션**
- **Asset Store 무료**:
  - `Crest Ocean System` (무료 버전 있음, AAA 품질)
  - `Stylized Water 2` 평가판 (만화풍 추천)
  - `Low Poly Water` 무료
- **직접 구현 (URP Shader Graph)**:
  - SeaPlane 머티리얼에 노멀맵 2개 (다른 스케일·속도)
  - UV 스크롤 (두 방향)
  - Fresnel 효과로 깊이감
  - 거품 라인 (해안 근처)

**작업 위치**: `Assets/Game/Art/Materials/Sea_Stylized.mat`

---

### B. 육지 컬러 팔레트 (1시간)
**현재**: `M3WorldMeshBaker.LandColor` 단일색 하드코딩.

**개선**
- ScriptableObject 로 분리: `LandColorPalette.asset`
- 위도별 그라데이션:
  - 극지 (lat |60~90|): 흰색 / 회색 (눈·얼음)
  - 한대 (lat |40~60|): 어두운 초록 / 침엽수림 톤
  - 온대 (lat |20~40|): 밝은 초록 / 황토
  - 열대 (lat |0~20|): 진한 초록 / 갈색
- 또는 대륙별:
  - 유럽 · 동아시아 → 초록 톤
  - 북아프리카 · 중동 → 모래 톤

**작업 위치**: `M3WorldMeshBaker.cs` + 새 `LandColorPalette.cs` SO

---

### C. 패널 카드형 배경 + 아이콘 (점진적, 패널당 30분)
**현재**: 회색 박스 + 텍스트만.

**개선 단위 (한 번에 하나씩)**
- 둥근 모서리 + 그림자 → `RoundedPanel` 컴포넌트 (이미 일부 있음, 확장)
- 텍스트 옆 이모지 / 아이콘
  - 🛒 시장 · ⚓ 조선소 · 👥 광장 · 🔧 수리 · 📋 명부
  - ⚔ 대포 · ✈ 속도 · 📦 화물 · 🛡 내구 · 💰 가격
  - (이미 일부 적용됨)
- 헤더 / 본문 구분선 (얇은 선 GameObject)
- 행마다 호버 / 짝수행 색 (zebra)

**대상 우선순위 (안정도 순)**
1. PlayerInfoPanel — 방금 만듦, 안정 → 디자인 폴리시 OK
2. CrewPanel — 안정
3. MarketPanel — 자주 보임, 안정
4. ShipyardPanel — 수리 통합 직후 → 며칠 사용 후 폴리시
5. PlazaPanel — 활동률 튜닝 중 → 안정 후 폴리시

---

### D. 항구 입구 아이콘 / 마커 (1~2시간)
**현재**: 항구 위치에 단순 큐브 / 원.

**개선**
- 항구별 깃발 아이콘 (국가 컬러)
- 항구 이름 라벨 (TMP World Space)
- 호버 시 도착 거리 표시

## 시각 폴리시 단발 vs 시스템

| 종류 | 작업 시간 | 임팩트 | 변경 비용 |
|---|---|---|---|
| 바다 셰이더 (Asset Store) | 1~2h | ★★★★★ | 낮음 |
| 육지 컬러 팔레트 | 1h | ★★★ | 낮음 |
| 패널 카드 디자인 | 패널당 30m | ★★ | 낮음 |
| 항구 마커 폴리시 | 1~2h | ★★★ | 낮음 |
| 배 3D 모델 (15종) | 종당 1~3일 | ★★★★★ | 중간 |
| 큰 지도 시스템 | 1~2일 | ★★★★ | 중간 (코드) |
| 게임일 시스템 | 2~3일 | ★★★ | 높음 (전 코드) |
| 사운드 풀세트 | 1주+ | ★★★★ | 낮음 |

→ **단발 시각 작업 (바다·육지·아이콘) 은 시스템 사이사이 1~2시간씩 끼워 넣기 좋음.**

## 우선 추천

1. **A. 바다 셰이더** (지금) — 가장 큰 시각 효과, 한 번 적용하면 끝
2. **C. 패널 폴리시** (점진적) — 안정된 패널부터 한 번에 하나씩
3. **B. 육지 팔레트** (다음 시스템 작업 사이) — 1시간 짧게
4. **D. 항구 마커** (큰 지도 작업과 함께) — 길잡이 시스템과 연결

이 순서대로 진행하면 시각 임팩트는 즉시 받으면서도 시스템 작업이 멈추지 않음.
