# 자연물 스트리밍 · 밀도 조절 가이드 (NatureStreamer)

플레이어가 맵 전체를 돌아다녀도 가볍게 자연물을 채우는 **스트리밍 방식**과,
**밀도(빽빽함)를 조절하는 방법**을 정리한 문서입니다.

- 스크립트: `Assets/Game/Scripts/NatureStreamer.cs`
- 모델: `Assets/Game/Art/Models/Nature/` (Tree_Pine, Tree_Round, Tree_Dead, Stump, Rock, Rock_Small, Bush, Flower, Grass_Tuft)

> 스트리밍 방식은 **맵 크기를 몰라도 됩니다.** 플레이어 주변만 만들고 멀어지면 지우므로,
> 맵이 아무리 커도 작동하고 화면에 보이는 개수가 항상 일정합니다.

---

## 1. 기본 설정

1. 프리팹 준비: Nature 폴더의 FBX를 씬에 올렸다가 `Assets/Game/Prefabs`로 드래그 → 프리팹화
2. Hierarchy에 빈 GameObject 생성 (이름: `NatureStreamer`)
3. `Add Component` → **Nature Streamer**
4. **Target** 칸에 **플레이어** 드래그 (비우면 Main Camera를 따라감)
5. **Prefabs**에 나무·바위·꽃 프리팹 채우기
6. **Min Height**를 수면 높이로 (바다엔 안 생기게)
7. 지형(육지)에 **Collider** 필요 (메쉬면 Mesh Collider)
8. Play → 플레이어 주변에 자연물이 따라 생김

## 2. 밀도(빽빽함) 조절 — 핵심

### 공식
```
밀도        ≈ Per Chunk ÷ (Chunk Size × Chunk Size)
화면 총 개수 ≈ Per Chunk × (2 × View Radius + 1)²
```

### 조절 방법

**방법 1 — Per Chunk 올리기 (가장 쉬움)**
- 한 구역당 개수. 키운 만큼 그대로 빽빽해짐
- 예: 40 → 80 → 150
- View Radius는 3 정도로 두고 Per Chunk만 올리는 게 안전 (총 개수가 덜 폭발)

**방법 2 — Chunk Size 줄이기 + View Radius 키우기**
- 같은 개수를 더 좁은 구역에 넣으면 빽빽해짐 (Chunk Size ↓)
- 구역이 작아지면 같은 거리 보려고 View Radius를 올려야 함
- 예: Chunk Size 50→30, View Radius 3→5

### 예시 세팅

| 느낌 | Chunk Size | View Radius | Per Chunk | 대략 총 개수 |
|------|-----------|------------|-----------|------------|
| 기본 | 50 | 3 | 40 | ~2,000 |
| 빽빽 | 50 | 3 | 120 | ~5,900 |
| 더 빽빽(가까이만) | 30 | 4 | 80 | ~6,500 |

## 3. 각 값 설명

| 항목 | 설명 |
|------|------|
| **Target** | 따라다닐 대상(플레이어). 비우면 Main Camera |
| **Prefabs** | 뿌릴 프리팹들 (받아둔 에셋도 가능) |
| **Chunk Size** | 한 구역 크기 |
| **View Radius** | 주변 몇 구역까지 채울지 (시야 거리). 클수록 멀리 보이나 무거움(4~5 이하 권장) |
| **Per Chunk** | 한 구역당 개수 = 밀도 |
| **Min/Max Height** | 이 높이 범위에만 배치 (Min을 수면으로 → 바다 제외) |
| **Max Slope** | 이 경사보다 가파르면 안 놓음(절벽 제외) |
| **Scale Range** | 크기 무작위 다양화 |
| **Max Tilt** | 개체별 랜덤 기울기(뻣뻣함 제거) |
| **Color Variation** | 개체별 밝기 변화(똑같음 제거) |
| **Max Chunk Loads Per Frame** | 한 프레임에 새로 만드는 구역 수(끊김 방지) |

---

## 4. 성능 주의

- 빽빽할수록 **총 개수가 늘어 무거워짐.** 위 표의 "총 개수"를 보며 조절.
  보통 5,000~6,000개까지는 괜찮고, 그 이상은 프레임을 확인하며 올릴 것.
- **Color Variation을 켜면** 개체마다 색이 달라 묶음처리(배칭)가 안 돼 더 무거움.
  아주 빽빽하게 갈 거면 Color Variation을 0으로.
- 끊김이 느껴지면 **Max Chunk Loads Per Frame**을 1~2로 낮추기.

## 6. 잔디(아주 빽빽한 것)

잔디처럼 수천~수만 포기로 빽빽하게 깔 거면 **개별 오브젝트 방식은 한계**예요(무거워짐).
이 경우는 **GPU 인스턴싱**으로 따로 처리해야 가볍습니다. 필요 시 전용 스크립트 요청.

---

## 7. 빠른 튜닝 순서

1. 우선 Per Chunk를 2~3배 올려 원하는 밀도 확인
2. 프레임이 버티는 선에서 멈추기
3. 너무 뻣뻣/똑같으면 Max Tilt·Color Variation 조정
4. 멀리까지 보이게 하려면 View Radius ↑ (단 무거워짐)
5. 빽빽하면서 가벼우려면 Chunk Size ↓ + Per Chunk 적당히
