# 지형 / 항해 통과 개선 계획

좁은 해협(영국해협·말라카 등) 에서 배가 자주 막히는 문제를 해결하기 위한 계획.

## 현재 지형 구조

| 레이어 | 역할 | 비고 |
|---|---|---|
| `LandmassData` ×12 (M1) | 위/경도 박스 12개 | axis-aligned. SE Asia 1개 박스로 다 묶임 |
| `WorldLand.mesh` (M3) | Natural Earth GeoJSON 베이크 정밀 메쉬 | 씬에 prefab 드래그해야 활성 |
| `WorldCarves` | "이 영역만 충돌 무시" 카브 | **현재 지브롤터 1개만 등록** |
| `ShipController.collisionCheckRadius` | 배 충돌 반경 (2 units) | 좁은 통로일수록 영향 큼 |

## 문제 원인

- **북유럽**: `Landmass_Europe` 박스 (위 46~52, 경 0~30) + `BritishIsles` 박스가 영국해협·스카게라크·발트해 입구를 모서리로 막음.
- **동남아**: 단일 `Landmass_SoutheastAsia` 박스 (위 -4~14, 경 98~112) 안에 말라카·순다·셀레베스 해 다 포함 → 다 막힘.

근본은 "박스 axis-aligned 근사 + 카브 부족".

## 1단계 — 현재 씬 모드 확인 (5분)

- Hierarchy 에 `WorldLand` 메쉬가 있는지 vs `LandmassPlacer` 가 박스 spawn 인지 확인
- Play → 북유럽 + 동남아 진입 시도 → 어디서 막히는지 좌표 로그
- 그 결과로 카브 위치 미세 조정

## 2단계 — 해결 방식 (셋 중 하나)

| 방식 | 작업량 | 효과 | 사용자 편집성 |
|---|---|---|---|
| **A. WorldCarves 확장** (배열에 10~15개 하드코드) | 30분 | 즉시 통과 가능 | 코드 편집 필요 |
| **B. 박스 세분화** (SE Asia 박스 → 5~6 조각) | 1시간 | 시각·충돌 모두 자연스러움 | M1Seeder 수정 |
| **C. CarveEditor 도구** (PortPlacementEditor 패턴) | 2시간 | Play 중 드래그로 카브 추가/이동/리사이즈 + SO 저장 | ⭐ Play 모드 시각 편집 |

**추천: C → A 데이터 채움 순서.**
- WorldCarves 를 정적 배열 → SO 배열 (`WorldCarveCatalog`) 로 옮김
- 핸들 = 카브 원 (중심 핸들 + 반경 핸들 또는 더블클릭 → 반경 입력)
- 만들어두면 앞으로 모든 통과 문제를 Play 중 드래그로 해결

## 4단계 — 효과 검증

- 시드 후 Play → NPC + 플레이어 둘 다 막혔던 항구 도착 가능 확인
- `NavRouter` 게이트웨이가 카브 영역 내에 들어가도록 좌표 일치 확인
- 막힘 잔존 시 카브 반경 +20~30km 또는 새 카브 추가

## 메모

- 카브는 "충돌만" 무시 — 시각 메쉬는 그대로 (어린이가 진짜 해안선을 봄)
- 카브가 육지 면적의 너무 큰 부분을 덮으면 항해 의미가 사라지므로 신중히
- 반경 100km 정도면 도시 1개 통과 가능 / 200km 면 작은 군도 통과
