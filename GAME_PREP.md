# 어린이 대항해 시대 — 준비 문서

> 어린이가 **대항해시대 II**를 모티브로 한 세계 항해를 통해 **세계 지리 / 지역 특산물 / 숨겨진 발견물 / 역사** 를 자연스럽게 배우는 교육용 게임.

이 문서는 본격 구현 전 정해두어야 할 사항과 자산 / 코드 구조를 모아둔다. 구체적인 게임 디자인(레벨, 시나리오, 발견물 목록 등)은 별도 디자인 문서에서 다룬다.

---

## 1. 비전 (확정)

| 항목 | 결정 |
|---|---|
| 장르 | 교육용 항해 어드벤처 |
| 모티브 | 코에이 『대항해시대 II』 |
| 타겟 연령 | **초등 저학년 (1~3학년)** |
| 시각 | **2D 베이스 + 약간의 3D (2.5D)** |
| 조작 | **직접 조종** (자동 길찾기 아님) |
| 시대 배경 | **실제 대항해시대 (15~17세기)** — 가상 세계 아님 |
| 플랫폼 | **모바일 / 태블릿 전용** (PC 빌드 안 함) |
| 1차 시연 항구 | **리스본 ↔ 세우타** |
| 교육 목적 | ① 세계 지리 ② 지역 특산물 ③ 숨겨진 발견물 ④ 역사 |

핵심 원칙: **시뮬레이션 깊이보다 접근성 우선.** 어린이가 5~10분 안에 첫 항해를 시작할 수 있어야 한다.

### 1.1 결정이 시스템에 끼치는 영향 (요약)
- **초등 저학년**:
  - 모든 텍스트는 받침 많은 한자어 / 어려운 외래어 회피. 발음하기 어려운 지명(예: "캘리컷")은 한글 음차 + 작은 해설.
  - 폰트는 크고 둥글게, 터치/클릭 타깃은 넉넉히.
  - 실패(좌초·길 잃음)에 페널티를 주지 않는다. "잠시 후 자동 복귀" 같은 부드러운 처리.
  - 한 발견물의 해설은 **2~4문장**을 넘기지 않는다.
- **직접 조종**:
  - 클릭 자동 항해 시스템은 **만들지 않는다**. 모든 이동은 플레이어 입력 기반.
  - 조작은 단순화: 좌/우 = 키 회전, 위 = 가속, 아래 = 감속. 마우스 보조는 옵션.
  - 약한 관성(스르륵 멈춤)으로 "배 같은 느낌"은 주되, 멀미 날 정도는 X.
  - 미니맵 + 항구/발견물 방향 표시기로 길을 잃지 않게 보조.
- **실제 대항해시대 (15~17세기)**:
  - 모든 항구·발견물·인물·사건은 사실 기반. 출처를 데이터에 기록.
  - 시대상 민감 주제(노예무역, 식민지화, 종교 갈등)는 회피하지 않되 **사실 중심·중립적 어조**로.
  - 시각 톤은 고지도·범선 일러스트 분위기. 너무 SF / 판타지로 가지 않는다.
- **모바일 / 태블릿 전용**:
  - 입력은 **터치 only** — 가상 조이스틱, 슬라이더, 큰 탭 버튼. 키보드/마우스 매핑 만들지 않음.
  - 렌더 파이프라인은 **`Mobile_RPAsset` / `Mobile_Renderer`** 사용 (`PC_*` 에셋은 빌드에서 제외).
  - **가로 모드 우선** (배가 좌우로 항해하는 게임 특성).
  - 안전 영역(Safe Area)을 UI에 반영. iPhone 노치/펀치홀, Android 제스처 영역 회피.
  - 다양한 해상도/aspect ratio 대응: `Canvas Scaler` 의 `Scale With Screen Size`, Reference Resolution 1920×1080 또는 2208×1242 정도로.
  - 성능 예산: 60fps 목표, 저사양 폰(예: 4년 전 안드로이드) 기준. 3D 폴리곤 수 / 텍스처 크기 / 동적 라이트 보수적으로.
  - 발열·배터리 고려: 정적인 항구 화면에서는 프레임 캡 또는 일시정지 가능한 구조.

## 2. 대항해시대 II에서 가져올 요소 (그리고 가져오지 않을 것)

원작은 좋은 모티브이지만 그대로는 너무 무겁다. 아래와 같이 **선택적으로 차용**한다.

### 가져올 요소
- **세계 지도 항해**: 전 세계 단일 지도 위에서 항구 간을 오가는 큰 그림.
- **항구**: 도착 시 메뉴가 열리고, 도시 일러스트 + 그 지역 정보 표시.
- **특산물 / 무역 (간소화)**: 각 항구의 대표 특산물을 보여주는 학습 카드. 1차에서는 "구매-판매로 돈 벌기" 보다 **"이 지역의 특산물을 만나봐!"** 에 가깝게.
- **발견물 시스템 (핵심 교육 장치)**: 특정 지점에 도달하거나 항구에서 의뢰를 받아 **랜드마크 / 동식물 / 역사적 사건 / 유적** 을 발견 → 발견 시 일러스트 + 해설 텍스트(어린이 눈높이) + "발견물 도감"에 등록.
- **항해일지 / 도감**: 발견한 항목과 방문한 항구를 모으는 컬렉션 UI. 진행도 가시화.

### 가져오지 않을 (또는 1차에서 빼는) 요소
- 복잡한 무역 시세 변동, 금리, 투자 시스템.
- 해전 / 함대 전투 (어린이 친화성과 충돌).
- 항해사 고용, 함대 관리.
- 항해 중 식량/물/괴혈병 관리 (스트레스 요소 제거).

→ 1차 구현은 **"세계지도 항해 + 항구 도착 + 발견물"** 트라이앵글에 집중.

## 3. 2.5D — 무엇을 어떻게 섞나

"2D 베이스 + 약간 3D" 를 구체화하기 위한 결정.

### 권장 방향
- **세계지도 항해 화면**: 카메라는 top-down에 가까운 각도(예: 60~75°)로 살짝 기울임. 지도는 큰 3D 평면 + 양각으로 약간 돌출된 산맥/대륙. 배는 **3D 로폴리 모델**로 키 회전과 약간의 기울기(파도 흔들림) 표현.
- **항구 화면**: 2D 일러스트 배경 + 2D UI 카드. 굳이 3D 안 써도 됨.
- **발견 이벤트**: 2D 일러스트 컷씬 + 카메라 살짝 줌/팬으로 입체감.

### 기술 선택
- **렌더 파이프라인**: 기존 URP 유지. URP는 2D/3D 혼합에 적합.
- **카메라**: Cinemachine 사용 권장 — 항해/항구/이벤트별 가상 카메라를 정의하고 부드럽게 전환.
- **스프라이트 vs 3D 모델**:
  - 배: 저폴리 3D (회전이 부드럽고 파도 흔들림 표현 쉬움).
  - 지도 위 항구 마커, UI: 2D 스프라이트.
  - 대륙: 2.5D 평면에 노멀맵 또는 약간의 메쉬 양각.
- **2D 라이트**: URP의 2D Renderer 또는 3D Renderer 위에 Sprite Lit 셰이더 — 처음에는 단순하게 가고, 분위기 잡을 때 도입.

→ **`Mobile_RPAsset` / `Mobile_Renderer` 기준** 으로 시작한다. 기존 `PC_*` 에셋은 참고용으로 두되 빌드에는 사용하지 않음. 향후 빌드 타깃은 Android / iOS (우선순위는 §10에서 결정).

## 4. 게임 루프 (1차 마일스톤 범위)

```
[세계지도] → 항구 선택/항해 → [항해 중] → 항구 도착 또는 발견물 트리거
    ↑                                              ↓
    └──────  [항구 화면] / [발견 이벤트] ←────────┘
                       ↓
                  [도감에 등록]
```

### 1차 마일스톤: "리스본에서 출발해 세우타까지 항해하고, 도중에 지브롤터 해협을 발견한다"

**왜 리스본 ↔ 세우타인가?**
- 1415년 포르투갈의 **세우타 정복**은 대항해시대의 실질적 시작점 — 게임의 첫 항해로서 역사적 의미가 깊다.
- 약 600km 의 짧은 항로 → 초등 저학년이 처음 항해하기에 부담 없는 거리.
- 사이에 **지브롤터 해협**이 있어 자연스러운 첫 발견물 위치.
- 지중해와 대서양의 경계 → 이후 지중해 / 대서양 두 방향으로 확장 가능.

**체크리스트**
- [ ] `Assets/Game/` 폴더 + `Game.asmdef` 생성
- [ ] 빌드 타깃을 Android (또는 iOS) 로 전환, `Mobile_RPAsset` 적용
- [ ] `World.unity` 씬: 이베리아 반도 + 북아프리카 일부가 보이는 2.5D 평면 + Cinemachine 카메라 (가로 모드)
- [ ] 배 3D 모델 (임시 캐러벨선 또는 저폴리 임시) + `ShipController` (터치 입력으로 조타/추진)
- [ ] 터치 UI: 가상 조이스틱(조타) + 가속/감속 슬라이더 또는 버튼
- [ ] 항구 트리거 2개: **리스본**(38.7°N, 9.1°W 근방), **세우타**(35.9°N, 5.3°W 근방)
- [ ] 발견물 트리거: **지브롤터 해협** (36.0°N, 5.6°W) + 발견 패널 UI (일러스트 + 2~4문장 해설)
- [ ] 도감 ScriptableObject + 발견 시 등록 + 도감 보기 UI (탭으로 열림)
- [ ] 항구 도착 화면: 임시 배경 + "리스본 / 포르투갈" "세우타 / 모로코" 라벨 + 특산물 한 줄
- [ ] 실기 1대(안드로이드 또는 아이패드)에 빌드 → 손가락으로 실제 조작감 확인

이 한 사이클이 굴러가면 그 위에 항구/발견물/지역을 **데이터(SO) 추가만으로** 늘려갈 수 있다.

## 5. 교육 콘텐츠 — 데이터로 다룰 것

게임 시스템과 별개로, 교육용 게임은 **콘텐츠 데이터의 양과 질**이 핵심이다. 처음부터 데이터 구조를 깔끔히 잡아두면 나중에 콘텐츠 추가가 코드 변경 없이 가능하다.

### 5.1 데이터 모델 (ScriptableObject로)
- **`PortData`**: 이름, 위/경도(또는 지도 위 좌표), 대표 도시 일러스트, 소속 국가/문화권, 대표 특산물 리스트, 한 줄 설명(어린이 눈높이).
- **`ProductData`** (특산물): 이름, 아이콘, 원산지(항구 참조), 짧은 해설("후추는 인도에서…").
- **`DiscoveryData`** (발견물): 이름, 카테고리(랜드마크/동식물/유적/사건), 발견 지점(좌표 또는 항구 근처), 일러스트, 해설 텍스트, 역사적 시대.
- **`RegionData`**: 여러 항구/발견물을 묶는 지역 (예: 지중해, 동아시아).

### 5.2 콘텐츠 작성 가이드라인 (별도 문서로 분리 예정)
- 텍스트는 **초등 저학년(1~3학년) 어휘 수준** 기준. 한 문장 짧게, 어려운 한자어/외래어 회피.
- 한 발견물·항구의 메인 해설은 **2~4문장**. 더 알고 싶으면 "더 보기" 로 별도 페이지.
- 역사 서술은 **연도·인물 암기보다 "왜 그런 일이 있었는지" 위주.**
- 종교/식민지/전쟁 등 민감 주제는 사실 기반으로 중립적 서술.
- 자료 출처를 메타데이터에 기록 (저작권 안전 / 추후 검증 용이).
- 시대 고증: 15~17세기 기준. 그 시대에 존재하지 않은 사물/지명/국가명은 사용 금지.

### 5.3 다국어
- 1차는 한국어 전용.
- 단, 모든 텍스트는 처음부터 **키-값 분리**로 작성 (예: `port.lisbon.name`, `port.lisbon.description`). ScriptableObject에 직접 한글을 박아도 되지만, 키 기반 로컬라이즈 시스템(Unity Localization 패키지)을 처음부터 도입하면 추후 영어/일본어 확장이 무료에 가까워진다.

## 6. 코드 / 폴더 구조

```
Assets/
├── Bitpurr Digital/         ← 건드리지 않음 (Editor Terminal)

├── Game/                    ← 게임 코드/에셋 루트
│   ├── Art/
│   │   ├── Map/             세계지도 텍스처, 양각 메쉬
│   │   ├── Ships/           배 3D 모델
│   │   ├── Ports/           항구 일러스트
│   │   ├── Discoveries/     발견물 일러스트
│   │   └── UI/              UI 이미지, 폰트
│   ├── Audio/               BGM, SFX
│   ├── Data/                ScriptableObject 인스턴스 (PortData 등)
│   │   ├── Ports/
│   │   ├── Products/
│   │   ├── Discoveries/
│   │   └── Regions/
│   ├── Prefabs/             Ship, Port, DiscoveryTrigger
│   ├── Scenes/              World.unity, Port.unity (또는 단일 씬 + 패널 전환)
│   ├── Scripts/
│   │   ├── Ship/            ShipController, ShipMovement
│   │   ├── World/           WorldMap, CameraRig, GeoCoordinate
│   │   ├── Ports/           PortTrigger, PortPanelUI
│   │   ├── Discovery/       DiscoveryTrigger, DiscoveryPanelUI, Journal
│   │   ├── Data/            PortData, ProductData, DiscoveryData, RegionData (SO 정의)
│   │   ├── Input/           Input wrappers
│   │   ├── UI/              HUD, 메뉴, 도감 UI
│   │   ├── Save/            진행도 저장 (방문 항구, 발견물 목록)
│   │   └── Game.asmdef
│   └── Settings/            게임 전역 설정 SO (속도, UI 톤 등)
└── Settings/                기존 URP 설정 유지 (PC_RPAsset 사용)
```

**`Game.asmdef`** 로 Editor Terminal 어셈블리와 분리. 추후 `Game.Tests.asmdef` 추가 가능.

## 7. 필요한 에셋 (1차 마일스톤 기준, 최소 임시본)

| 카테고리 | 항목 | 1차에서 필요한 양 | 비고 |
|---|---|---|---|
| 지도 | 세계지도 텍스처 | 1장 | 임시는 위키미디어 commons의 자유 이용 지도라도 OK |
| 배 | 3D 저폴리 모델 | 1척 | Unity Asset Store / Sketchfab CC0 |
| 항구 | 도시 일러스트 | 2개 | 임시는 placeholder 색 박스 |
| 발견물 | 일러스트 | 1개 | 임시는 placeholder |
| UI | 한글 폰트 (라이선스 확인) | 1종 | Pretendard, 나눔 등 |
| BGM/SFX | — | 1차에서는 생략 가능 | |

**저작권**: 어린이용/배포 의도라면 처음부터 CC0 또는 명확한 상업 라이선스만 사용. 출처를 `Assets/Game/Art/CREDITS.md` 에 누적 기록.

## 8. 조작 / 입력 (확정안)

**직접 조종**으로 결정됨. 클릭 자동 항해 시스템은 만들지 않는다.

- **세계지도 화면 (항해 중)**:
  - `좌/우 방향키 (또는 A/D)` — 키 회전.
  - `위 방향키 (또는 W)` — 가속.
  - `아래 방향키 (또는 S)` — 감속 / 정지.
  - `Space` — 항구 도착 시 입항 / 발견물 트리거 확인.
  - `Tab` 또는 `J` — 항해일지(도감) 열기.
  - `M` — 큰 지도(미니맵 확장).
- **항구/발견 패널 UI**: 마우스 클릭 또는 `Space` / `Enter` 로 진행.
- **조작 보조 (초등 저학년 배려)**:
  - 약한 관성. 멈춤이 너무 길어지지 않도록 자연 감속을 추가.
  - 미니맵에 가까운 항구/발견물 방향 화살표 표시.
  - 육지에 부딪혀도 **데미지/사망 없이** 살짝 튕기고 끝.

### 8.1 Input Actions 재작성
기존 `Assets/InputSystem_Actions.inputactions` 의 `Move` / `Look` / `Attack` / `Interact` / `Crouch` 는 이 게임에 맞지 않으므로 액션 맵을 다시 만든다.

| 새 액션 | 타입 | 바인딩 (PC) |
|---|---|---|
| `Steer` | Value (float, -1~1) | A/D, ←/→ |
| `Throttle` | Value (float, -1~1) | W/S, ↑/↓ |
| `Confirm` | Button | Space, Enter, 마우스 좌클릭 |
| `OpenJournal` | Button | Tab, J |
| `OpenMap` | Button | M |
| `Cancel` | Button | Esc, 마우스 우클릭 |

## 10. 여전히 사용자에게 묻고 싶은 것

> ✅ 결정됨: 연령대(초등 저학년), 조작(직접 조종), 시대 배경(실제 15~17세기).

남은 디테일:

1. **플랫폼**: PC만 가도 되는가? 향후 태블릿/모바일 고려가 필요한가?
2. **세션 길이**: 한 번 켰을 때 5~10분에 끝나는 미니 항해 위주인가, 한 시간 이상 이어지는 모험인가?
3. **1차 시연용 항구 후보**: 리스본 / 카이로 / 베네치아 / 이스탄불 / 캘리컷 등 — 어디서 시작하면 좋을까? (1차 마일스톤은 항구 2개)
4. **출발 항구 / 주인공 국적**: 대항해시대2처럼 여러 국적 중 고르는가, 아니면 고정인가?
5. **개발 인력**: 사용자 본인 단독? 아트는 외주/에셋스토어? 텍스트 콘텐츠는 누가 작성?
6. **로컬라이즈 시스템**: 처음부터 Unity Localization 패키지를 깔고 갈 것인가, 일단 한글 하드코딩으로 빨리 갈 것인가?
